# 执行计划

## Phase 0：准备环境

目标：让 Windows 本机和 Ubuntu 虚拟机都具备开发/部署基础。

Windows 本机：
- 确认 Node.js LTS 可用。
- 确认 Git 可用。
- 项目路径保持为 `D:\Project\Web\JPLearn`。
- 学习资料路径保持为 `D:\学习\日语`。

Ubuntu 虚拟机：
- 安装 Ubuntu Server 22.04 LTS 或 24.04 LTS。
- 启用 OpenSSH Server。
- 配置桥接网络或保证 Windows 可 SSH 访问。
- 安装 Docker 和 Docker Compose。

验收：
- Windows 可以运行 `node --version`、`npm --version`。
- Windows 可以 `ssh 用户名@虚拟机IP`。
- 虚拟机可以运行 `docker --version` 和 `docker compose version`。

## Phase 1：初始化 Web 项目

目标：建立 Next.js + TypeScript + Tailwind + Prisma 基础项目。

任务：
- 初始化 Next.js App Router 项目。
- 安装 Prisma、SQLite、OpenAI SDK、必要 UI 依赖；AI 配置按 DeepSeek v4 flash 的 OpenAI 兼容接口预留。
- 建立基础页面：首页、练习页、知识点列表页、设置页。
- 建立 `.env.example`。
- 建立基础 Dockerfile 和 compose 文件。

验收：
- `npm run dev` 可启动。
- 首页可访问。
- `npm run build` 通过。

## Phase 2：数据模型与导入

目标：把现有 N5 词汇和语法资料导入 SQLite。

任务：
- 编写 Prisma schema。
- 创建迁移。
- 编写导入脚本读取两个 CSV。
- 导入 644 个词汇知识点和 84 个语法知识点。
- 建立掌握状态初始化逻辑。

验收：
- 数据库内共有 728 个知识点。
- 词汇和语法数量分别正确。
- 列表页能显示导入数据。
- 随机抽查 5 个词汇和 5 个语法字段映射正确。

## Phase 3：知识点浏览

目标：完成无 AI 依赖的核心浏览能力。

任务：
- 实现知识点列表接口。
- 实现知识点详情接口。
- 实现筛选和搜索。
- 展示掌握状态。
- 支持手动重置掌握状态。

验收：
- 可以按词汇/语法筛选。
- 可以按已掌握/未掌握筛选。
- 可以打开详情页查看完整信息。
- 无 AI 配置时页面仍可用。

## Phase 4：AI 例句生成

目标：为知识点生成并缓存例句。

任务：
- 建立 AI 客户端封装。
- 实现例句生成 prompt。
- 实现结构化 JSON 解析和错误处理。
- 实现例句缓存。
- 在详情页和练习页展示例句。

验收：
- 对任意知识点可生成 5 条例句。
- 第二次打开同一知识点优先使用缓存。
- AI 失败时显示错误并允许重试。

## Phase 5：练习批改与掌握度

目标：实现完整学习闭环。

任务：
- 实现练习抽题逻辑：未掌握优先，兼顾最近错误。
- 实现知识点到题型的映射逻辑，优先覆盖文本题。
- 实现用户理解题和造句题提交。
- 实现 AI 批改 prompt。
- 保存练习提交记录和逐知识点评估明细。
- 按评估明细逐项更新掌握状态。
- 对应知识点理解或使用正确加 20 分、错误减 10 分，达到 100 分后标记掌握。

验收：
- 用户可以完成一次练习。
- 正确/错误反馈和每个知识点的加减分明细能保存并展示。
- 掌握分数达到 100 后知识点变为已掌握。
- AI 调用失败不会更新掌握状态。

## Phase 6：虚拟机部署

目标：验证 Linux 部署路径。

任务：
- 把项目同步到 Ubuntu 虚拟机。
- 配置 `.env`。
- 用 Docker Compose 启动应用。
- 挂载 SQLite 数据目录。
- 在 Windows 浏览器访问虚拟机地址。

验收：
- 虚拟机上应用可启动。
- Windows 可访问页面。
- 数据库文件在容器重建后不丢失。
- AI 配置可用时能生成例句和批改。
