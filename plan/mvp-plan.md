# 日语学习工具 MVP 计划

## Summary

构建一个可本机运行、可迁移到 Linux 的 Web 学习工具。第一版采用“本机单用户 + DeepSeek v4 flash/OpenAI 兼容 API + 阅读例句/理解或造句批改”闭环：导入现有 N5 词汇和语法资料，把每个词汇/语法都作为知识点，AI 生成例句，用户完成理解或造句练习，AI 批改，并用 0-100 分的掌握分数追踪每个知识点。

默认不需要先装虚拟机。本机用 Node.js 跑开发环境；之后用 Docker 迁移到 Linux 服务器。

## Key Changes

- 技术栈：
  - 前端/后端一体：Next.js App Router + TypeScript。
  - UI：Tailwind CSS，先做桌面和移动端都可用的简洁学习界面。
  - 数据库：SQLite + Prisma，便于本机测试；保留后续切 PostgreSQL 的路径。
  - AI：暂定 DeepSeek v4 flash，通过 OpenAI 兼容接口接入，并用 `.env` 配置 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL`。
  - 部署：提供 Dockerfile 和 compose 配置，Linux 服务器可直接部署。

- 数据导入：
  - 从 `D:\学习\日语\词汇\JLPT_N5_常用核心词汇表.csv` 导入 644 个词汇知识点。
  - 从 `D:\学习\日语\语法\日语N5语法知识点大纲.csv` 导入 84 个语法知识点。
  - TXT 说明文件暂不作为数据源，因为读取存在编码乱码；Markdown 只作为参考资料保留。

- 数据模型：
  - `KnowledgePoint`：统一存词汇和语法，字段包括类型、标题、读音、分类、释义、接续/词性、原始例句、备注、来源编号。
  - `GeneratedExample`：缓存 AI 生成的例句，关联知识点，避免重复花费。
  - `PracticeAttempt`：记录用户练习答案、整体反馈、纠错说明、模型原始返回、时间。
  - `PracticeReviewItem`：记录一次练习中每个受影响知识点的正确/错误、加减分和说明。
  - `MasteryState`：记录每个知识点的掌握分数、正确次数、错误次数、是否掌握、最近练习时间。
  - `UserProfile`：第一版固定创建一个本地用户，后续可扩展多用户。

- 学习流程：
  - 首页展示进度：总知识点、已掌握、待练习、今日练习数。
  - 知识点列表支持按“词汇/语法/已掌握/未掌握/分类”筛选。
  - 知识点详情展示释义、读音、词性或接续、原始例句、AI 例句。
  - 练习页按未掌握优先抽取知识点，展示提示和例句，用户完成理解题或造句题。
  - AI 按知识点逐项批改，返回整体反馈、问题说明、修改建议、自然表达和每个知识点的正确/错误明细。
  - 每个知识点使用 0-100 分掌握分数；对应知识点理解或使用正确加 20 分，错误减 10 分，最低 0 分，达到 100 分后标记为已掌握。
  - 已掌握知识点仍可手动复习或重置掌握状态。

- AI 行为：
  - 例句生成：每次为单个知识点生成 5 条 N5 难度日语例句，附中文释义，结果入库缓存。
  - 批改：要求 AI 至少判断目标知识点，并对答案中明显涉及的词汇/语法逐项判断；理解和造句都没有部分正确，只有正确或错误；输出结构化 JSON。
  - 若 AI 调用失败，界面显示失败原因，并允许重试；不会更新掌握度。

## Test Plan

- 数据导入测试：
  - 确认导入后词汇为 644 条、语法为 84 条。
  - 抽查词汇字段：`word/kana/type_zh/zh/english` 正确进入数据库。
  - 抽查语法字段：`Grammar/Category/Pattern/Meaning/Example/Note` 正确进入数据库。

- 功能测试：
  - 能浏览知识点列表并筛选词汇/语法。
  - 能打开知识点详情并生成/复用 AI 例句。
  - 能提交造句并保存批改结果。
  - 同一知识点掌握分数达到 100 后显示已掌握。
  - AI 失败时不改变掌握状态。

- 部署测试：
  - Windows 本机 `npm run dev` 可访问。
  - `npm run build` 通过。
  - Docker 镜像能启动并使用 SQLite 数据文件。
  - `.env` 缺失 AI key 时，应用仍可浏览知识点，但生成和批改按钮提示配置缺失。

## Assumptions

- 第一版按本机单用户设计，不做登录、注册、权限、限流和公开服务防刷。
- 第一版暂定使用 DeepSeek v4 flash；代码层保持 OpenAI 兼容 API 抽象，方便后续切换供应商或模型。
- 第一版以“阅读例句 + 理解/造句 + AI 批改”为核心，不先做 Anki 式卡片。
- SQLite 足够支撑 N5 资料和个人学习记录；多人公开使用前再迁移 PostgreSQL。
- 资料 CSV 是权威导入源，Markdown/TXT 暂不参与自动解析。
